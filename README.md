# FastAPI Test App

## Описание и бизнес-логика

Это тестовое приложение на **FastAPI**, предназначенное для демонстрации системы пользователей с аутентификацией и авторизацией.  

Основная идея: есть несколько ролей пользователей, каждая с разными правами доступа к функционалу приложения.  

### Пользователи

В системе есть три типа пользователей:

- **Admin** – имеет полный доступ ко всем ресурсам приложения.
- **Manager** – доступ к ограниченному набору функций, управляет определенными ресурсами.
- **User** – базовый пользователь, доступен минимальный функционал.

При запуске приложения автоматически создаются seed-пользователи:  

| Email                | Роль    | Пароль       |
|---------------------|---------|--------------|
| admin@example.com    | admin   | admin123     |
| manager@example.com  | manager | manager123   |
| user@example.com     | user    | user123      |

### Система аутентификации

- Пользователи аутентифицируются через **JWT-токены**.
- Пароли хранятся в базе в виде **argon2-хэшированного** значения.
- Токен создаётся при успешной авторизации и используется для последующих запросов к защищённым эндпоинтам.

### Система авторизации (Policies)

- Роли пользователей определяют их доступ к ресурсам.
- **Admin** – полный доступ ко всем эндпоинтам.
- **Manager** – ограниченный доступ к управлению определёнными ресурсами.
- **User** – доступ только к своим данным.
- Проверка ролей реализована через **policy-декораторы** на эндпоинтах.

## Запуск приложения

### Предварительные требования

- Docker и Docker Compose
- Python 3.12 (для локального запуска без Docker)
- PostgreSQL (если запуск локально без Docker)

### Запуск через Docker
1. Запустите приложение и базу данных:

```bash
docker-compose up --build
```

2. После старта автоматически создаются seed-пользователи, и сервер будет доступен на:

```bash
http://localhost:8000
```

3. Для проверки работы:

Эндпоинт /docs – Swagger UI с документацией API.
```bash
http://localhost:8000/docs#/
```

Используйте email и пароль seed-пользователей для получения Access Token.
```
POST http://127.0.0.1:8000/api/v1/sessions/login

json body:
{
	"email": "admin@example.com",
	"password": "admin123"
}
```
